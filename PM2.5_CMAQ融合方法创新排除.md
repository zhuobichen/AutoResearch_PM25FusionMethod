# PM2.5 CMAQ融合方法创新排除规则

生成时间：2026-04-07

---

## 一、创新方法溯源要求（推荐填写）

### 1.1 推荐标明的内容

每个创新方法推荐明确说明：
- **参考论文**：基于哪篇论文的方法（如有）
- **参考方法**：基于哪个已有方法（如VNA、eVNA、GPR等）
- **创新点**：与参考方法的核心差异
- **物理动机**：为什么这个改进是合理的

### 1.2 溯源格式（推荐填写）

```markdown
## 创新方法溯源

| 项目 | 内容 |
|------|------|
| 方法名称 | xxx |
| 参考论文 | 《论文标题》/ 无 |
| 参考方法 | VNA / eVNA / GPR / 克里金 / 无 |
| 核心创新 | ... |
| 物理动机 | ... |
```

**注**：如果方法是自己创新提出没有直接参考来源，可填"无"，但必须明确物理动机。

## 二、需要排除的方法类型

### 2.1 加权集成方法（Stacking/Ensemble with learned weights）

**定义**：将多个基础方法预测值作为特征，用机器学习方法（Ridge/Lasso/线性回归等）学习最优权重，然后加权融合。

**典型代表**：
- SuperStackingEnsemble
- EnhancedStackingEnsemble
- UltimateStackingEnsemble
- FeatureStackingEnsemble
- MultiLevelStackingEnsemble
- AdaptiveOnlineEnsemble
- LogRatioEnsemble
- V6-Ensemble-V1
- OptimizedTripleEnsemble
- AdaptiveTripleEnsemble
- TripleEnsemble
- DualBiasFusion
- RefinedBayesianFusion
- BayesianInspiredFusion
- CorrelationGuidedFusion
- KernelAwareFusion
- CMAQConditionedFusion

**公式特征**：
```python
# 典型的加权集成
X_meta = [method1_pred, method2_pred, method3_pred, ...]
meta = Ridge(alpha=xxx).fit(X_meta, y_true)
final_pred = meta.predict(X_meta)
# 或
final_pred = w1 * method1 + w2 * method2 + w3 * method3 + ...
```

### 1.2 排除原因

#### 1.2.1 迁移性能不强
- 权重是在特定日期数据上学习得到的
- 不同日期可能需要不同的权重
- 学习到的权重可能过拟合于训练数据

#### 1.2.2 效果提升有限
- 往往比最优子方法提升很小（<0.005）
- 甚至可能不如最优子方法
- 例如：SuperStackingEnsemble(R²=0.8571) < RK-Poly(R²=0.88+)

#### 1.2.3 可解释性差
- 负权重的含义不明确
- 权重组合的物理意义不清楚

#### 1.2.4 复杂但无用
- 实现复杂，参数众多
- 实际应用中难以维护和部署

## 二、正确的创新方向

### 2.1 有效的创新类型

**有效的创新**应该是：
1. **有明确参考来源**（论文/方法）
2. **可解释的物理/数学模型**
3. **单模型就能超越基准**
4. **可迁移到其他日期**

### 2.2 鼓励的创新方向

| 类型 | 示例 | 参考来源 | 物理意义 |
|------|------|---------|---------|
| 新偏差校正 | RK-Poly | eVNA + 多项式回归 | CMAQ偏差非线性 |
| 新空间建模 | GPR降尺度 | 地统计学 | 残差空间相关性 |
| 新融合策略 | 协同克里金 | 地统计学 | 多变量空间协同 |

---

## 三、创新方法判定标准

### 3.1 排除条件

满足以下任一条件的方法应被排除：

1. **使用Ridge/Lasso/线性回归学习权重**
2. **多个基础方法预测值简单加权求和**
3. **依赖"负权重"消除模型间相关性**
4. **无物理意义的调参公式**

### 3.2 创新方法判定流程

```
1. 新方法是否使用权重学习（Ridge/Lasso等）？
   → 是 → 排除
   → 否 → 继续

2. 新方法是否有物理可解释性？
   → 否 → 排除
   → 是 → 保留（创新成立）

[推荐填写] 参考来源：最好标明参考论文/方法，便于溯源写文章
```

## 四、典型案例

### 4.1 案例1：SuperStackingEnsemble（应排除）

| 属性 | 值 |
|------|-----|
| R² | 0.8571 |
| 最优子方法(RK-Poly) R² | 0.8519 |
| 差值 | +0.0052（统计不显著） |
| 结论 | **应排除** |

**原因**：
- Ridge权重组合无物理可解释性
- +0.0052差距可能是运气，统计不显著
- 权重对单日数据敏感，跨日期不稳定

### 4.2 案例2：RK-Poly（应保留）

| 属性 | 值 |
|------|-----|
| R² | 0.8519 |
| 方法类型 | 单模型 |
| 可迁移性 | 是 |
| 可解释性 | 是 |
| 结论 | **应保留** |

**原因**：物理意义清晰（多项式校正 + GPR空间插值），单模型效果好。

## 五、总结

### 5.1 排除的方法特征

| 特征 | 描述 |
|------|------|
| 权重学习 | 使用Ridge/Lasso等学习权重 |
| 简单加权 | final = Σ(w_i * pred_i) |
| 负权重 | 通过负权重消除相关性 |
| 无显著提升 | R²提升 < 0.02 |

### 5.2 保留的方法特征

| 特征 | 描述 |
|------|------|
| 单模型 | 单一方法解决问题 |
| 物理可解释 | 数学/物理意义清晰 |
| 显著提升 | R²提升 ≥ 0.02 |
| 可迁移 | 权重不随日期剧烈变化 |
