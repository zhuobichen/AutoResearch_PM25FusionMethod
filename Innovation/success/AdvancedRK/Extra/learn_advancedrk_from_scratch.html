第一章

认识PM2.5：什么是空气质量监测？

先理解我们正在解决的问题是什么。

🌬️什么是PM2.5？

PM2.5是指直径小于2.5微米的颗粒物（如灰尘、烟雾、酸雨颗粒等）。它能深入肺部，进入血液，导致哮喘、心脏病、肺癌等健康问题。

  • PM2.5浓度越高，空气质量越差，对健康危害越大
  • PM2.5的浓度单位是 μg/m³（微克/立方米）
  • 世界卫生组织建议：日均浓度应低于25 μg/m³

📍如何监测PM2.5？

目前有两种主要方式：

🏭 CMAQ模型
|
🛰️ 卫星遥感
|
🏠 地面监测站

两种监测方式各有优缺点

方式优点缺点
地面监测站数据精确（专业仪器测量）站点稀少，全国只有几百个
CMAQ模型覆盖全国，每个位置都能算计算结果有误差，和真实值不完全一致

想象一下：地面监测站就像是用精密天平称重量（准确但只有几个点），而CMAQ模型就像是用尺子估算重量（有系统性误差但覆盖全区域）。

🎯

核心问题：我们能不能结合两者的优点——用地面监测站的精确数据来校正CMAQ模型的结果，得到又准确又完整的PM2.5分布图？

第二章

CMAQ模型：它是怎么工作的？

理解为什么CMAQ模型的结果会有误差。

🔬什么是CMAQ模型？

CMAQ（Community Multiscale Air Quality）是美国环保署开发的大气化学传输模型。它模拟大气中化学物质的产生、传输、转化和清除过程，从而计算出PM2.5的浓度。

🔧 打个比方

想象你用天气预报模型预测明天的温度。它需要输入：今天的温度、风速、湿度、气压...然后通过复杂的物理公式计算。模型越准确，预测越接近真实。

  • 输入：排放源数据、气象条件、地形等
  • 处理：大气化学动力学方程、污染物传输方程
  • 输出：每个网格点的PM2.5浓度预测值

⚠️CMAQ的误差来源

模型结果和真实值之间总会有差距，主要原因：

  • 排放源不确定性：工厂、汽车、扬尘...真实的排放量很难精确统计
  • 气象模拟误差：风场、温度场的微小误差会放大
  • 化学机理简化：真实的大气化学极其复杂，必须简化
  • 地形影响：山谷、高楼的局地影响难以精确模拟
📊

关键发现：CMAQ的误差不是随机噪声，而是有规律可循的。例如：低浓度时可能高估，高浓度时可能低估。

🗺️CMAQ的网格输出

CMAQ把全国划分成网格（每个格子约4km×4km），每个格子都有一个预测值。但监测站只有几百个，如何用这几百个准确数据来校正全图的预测值？这就是数据融合要解决的问题。

第三章

数据融合：我们想要什么？

理解"融合"的目标和挑战。

🎯数据融合的目标

我们想要一张完整又准确的PM2.5地图：

  • 准确：每个点的值都要接近真实值（参考地面监测站）
  • 完整：不是只有监测站附近准，而是全国每个地方都要准
💡

类比：你想知道明天全国每个城市的温度。监测站提供的是"真实温度"，但城市太少；CMAQ模型提供的是"预测温度"，覆盖全但有误差。融合就是：用真实温度来校正预测温度。

⚠️融合的核心困难

困难在于：CMAQ的偏差不是常数，而是与多种因素有关：

  • 偏差和CMAQ预测值本身可能呈非线性关系（低浓度高估、高浓度低估）
  • 偏差在不同位置有不同的空间结构（顺风方向影响更远）
  • 不同季节的偏差规律可能不同

AdvancedRK的解决方案：用两步分工来处理这两个问题。

🚀AdvancedRK的思路：两步走

1

第一步：多项式校正

处理"CMAQ偏差和浓度的关系是非线性的"这个问题

2

第二步：GPR空间建模

处理"校正后仍然有局部空间差异"这个问题

🔑

核心洞察：与其用一个复杂模型同时处理两个问题，不如用两个简单的专业模型各司其职。

第四章

基准方法：VNA / eVNA / aVNA / Downscaler

理解AdvancedRK对比的基准方法是什么，它们各自有什么特点。

🗺️VNA（Voronoi Neighbor Averaging）

核心思想：对每个预测点，找离它最近的k个监测站，用距离加权平均估算值。

VNA(s) = Σᵢ wᵢ(s) · Oᵢ / Σᵢ wᵢ(s) 其中 wᵢ(s) = dᵢ(s)⁻²（距离的平方反比）

关键参数：

  • k：近邻数量（默认30，EPA标准）
  • power：距离权重指数（默认-2，即1/d²）
  • method：邻居选择方法（nearest/voronoi/laplace）
🧭 形象理解

就像在一张地图上，你站在某个位置，最近的30个朋友告诉你他们那边的温度，你根据距离远近加权平均，猜出你这里的温度。

局限：VNA直接用监测站的观测值，不利用CMAQ模型的预测值。所以当预测点远离监测站时，预测质量会急剧下降。

📐eVNA（enhanced VNA，比例校正）

核心思想：假设CMAQ模型值和真实值之间存在比例关系，用监测站的比值来校正。

r = O_monitor / O_model（在校正监测站处） eVNA(s) = O_model(s) × r（推广到全网格）

代码实现（来自core.py）：

@staticmethod
def compute_evna(model_values, vna_rn):
    return model_values * vna_rn
⚠️

局限：假设比例r是常数。但如果CMAQ在低浓度时偏高10%，高浓度时偏高20%，这个假设就不成立。

➕aVNA（additive VNA，加法校正）

核心思想：假设CMAQ模型值和真实值之间存在固定的偏移量，用监测站的偏差来校正。

b = O_monitor − O_model（在校正监测站处） aVNA(s) = O_model(s) + b（推广到全网格）

代码实现（来自core.py）：

@staticmethod
def compute_avna(model_values, vna_bias):
    return model_values + vna_bias
⚠️

局限：假设偏差b是常数。但如果CMAQ在北京偏差+5，在上海偏差-3，这个假设就不成立。

🎲Downscaler（MCMC downscaling）

核心思想：用MCMC（马尔可夫链蒙特卡洛）方法迭代校正模型输出。

1

初始化

用CMAQ模型值作为初始猜测

2

迭代校正

根据监测站数据计算偏差，逐步调整网格值

3

收敛

迭代2500次后得到稳定结果

🔧 打个比方

想象你调收音机：先随便转到一个频道（初始值），然后根据听到的清晰度微调旋钮（迭代校正），直到找到最清晰的频道（收敛）。

特点：需要全网格CMAQ数据作为输入，输出全网格校正结果，计算复杂度高，但能捕捉复杂的空间结构。

📊四种方法对比总结

方法偏差假设空间建模输出范围
VNA无（直接用监测站值）k近邻IDW加权全网格
eVNA比例常数（r_n）k近邻IDW加权全网格
aVNA偏移常数（bias）k近邻IDW加权全网格
Downscaler迭代学习（非线性）MCMC空间建模全网格
AdvancedRK多项式（非线性）GPR-Matern核全网格
🔑

核心区别：AdvancedRK用二阶多项式捕捉CMAQ偏差的非线性关系（不是常数比例或偏移），用GPR-Matern核做精细的空间建模（不是简单的距离加权）。

第五章

线性回归基础：理解多项式校正的原理

理解什么是"拟合"和"校正"——AdvancedRK第一步的数学基础。

📈什么是线性回归？

假设你有一些数据点，你想找到一条"最适合"的直线来描述它们的关系。

y = α₀ + α₁·x
例子：如果 x=50（PM2.5的CMAQ预测值）
那么预测的 y = α₀ + α₁×50

但真实值可能是 y = 55（监测站的值）

回归的任务：找到最佳的 α₀ 和 α₁，使得预测值和真实值最接近。

📊从1阶到2阶：多项式的升级

1阶（线性）：y = α₀ + α₁·x

📈 一条直线

2阶（二次）：y = α₀ + α₁·x + α₂·x²

📈 一条曲线（抛物线）
可以弯腰！
🎯 形象理解

想象你在拟合一个U形曲线（两头高中间低）。如果只用直线，永远拟合不好；但如果允许弯曲，就能很好地拟合。

⚠️为什么不能无限增加阶数？

理论上，3阶、4阶、10阶多项式可以拟合更复杂的曲线。但有一个问题：过拟合——曲线穿过每个点，但在新数据上表现很差。

消融实验证明：有GPR时，多项式阶数的影响很小（差异<0.003），二阶是最佳平衡点。

🔧多项式校正的代码逻辑

# 输入：CMAQ预测值 m，真实值 y
poly = PolynomialFeatures(degree=2)
m_poly = poly.fit_transform(m.reshape(-1, 1))  # 变成 [m, m²]
ols = LinearRegression().fit(m_poly, y)
y_poly = ols.predict(m_poly)  # 多项式预测
residual = y - y_poly  # 计算残差

简单说：找到一条最佳曲线，描述"CMAQ预测值"和"真实值"之间的非线性关系。

第六章

空间插值基础：为什么需要GPR？

理解什么是"距离加权"和"空间相关性"——AdvancedRK第二步的物理基础。

📍什么是空间插值？

你知道了A点和B点的值，想估算C点的值。这就是空间插值。

A点
浓度=50
←
C点（要估算）
→
B点
浓度=70

C点应该接近50还是70？取决于距离和方向。

⚖️最简单的插值：距离反比加权（IDW）

核心思想：离目标点越近的点，对目标点影响越大。

估算值 = Σ( 监测值_i / 距离_i ) / Σ( 1 / 距离_i )

VNA/eVNA/aVNA都使用这种IDW加权方式。优点是简单，缺点是没有考虑"空间相关性"——它假设所有方向都一样，但实际上PM2.5扩散有方向性（随风向）。

🌬️为什么需要GPR？（高斯过程回归）

IDW的问题：它假设"距离越近越像"，但实际情况更复杂：

  • 污染物扩散有方向性（顺风方向影响更远）
  • 地形会改变扩散路径（山谷、河谷）
  • 不同季节扩散规律不同

IDW就像是用"所有人的平均身高"来估算某个人的身高——考虑了距离，但没有考虑其他可能相关的因素。

💡

GPR的优势：它能学习数据的"相关性结构"——不是简单地说"越近越像"，而是说"在这个方向上相关性衰减更慢，在那个方向上衰减更快"。

🔗核函数的作用

GPR用核函数来描述"空间相关性"的结构。核函数就像测量两个点之间"有多像"的尺子。

RBF核
快速衰减
→ 超过某距离就"不像"
Matern核
平缓衰减
→ 逐渐远离，逐渐"不像"

Matern核的平缓衰减更符合PM2.5扩散的物理规律：污染物不会在某处突然从"有影响"变成"无影响"，而是逐渐衰减。

第七章

高斯过程回归（GPR）：它能做什么？

理解为什么GPR是空间建模的好工具。

📊GPR的基本思想

核心假设：我们想预测的"残差"（校正后的误差）服从某种空间相关的随机过程。

🎲 打个比方

想象你在一片草地上扔了一堆小球，每个小球落地后会留下一个"高度"。如果你在几个位置知道了小球的高度，你能不能猜测其他位置的高度？GPR的答案：可以！相邻的位置高度应该差不多，但有一定随机性。这叫"空间平滑"。

R*(s*) = μ + kᵀK⁻¹(R − μ) 其中： • μ：残差的均值（通常取0） • kᵢ = κ(s*, sᵢ)：新位置s*与训练点sᵢ的核函数值 • Kᵢⱼ = κ(sᵢ, sⱼ)：训练点之间的核函数矩阵

✨GPR的两大优势

1

预测 + 不确定性估计

不只是给一个数字，而是同时告诉你"这个数字有多可靠"

2

自动学习核函数参数

不需要手动设置"距离衰减有多快"，数据会告诉你

# GPR的输出
pred_mean, pred_std = gpr.predict(X_test, return_std=True)

# pred_mean: 预测值
# pred_std: 标准差（不确定性）
# 标准差大 → 预测不可靠（附近监测站稀疏）
# 标准差小 → 预测可靠（附近监测站密集）

⚙️稀疏矩阵：加速计算的关键

标准的GPR复杂度是 O(n³)（n是训练点数），如果n很大，计算会非常慢。

稀疏近似的思想：

  • 核函数 κ(d) 描述"距离为d的两个点有多像"
  • 当距离很远时，κ(d) ≈ 0（几乎不像）
  • 设定一个阈值，小于阈值的直接当0处理
稠密矩阵（1000×1000）
100万个数字
计算量：O(n³)
稀疏矩阵（~1%密度）
只有1万个数字不是0
计算量：大幅降低

以SparseKrig为例：设定阈值0.01后，核矩阵只保留约0.45%的非零元素。远距离的点对之间相关性几乎为0，直接当0处理，完全合理。

💡

类比：想象你在做一个1000×1000的拼图（100万块）。稀疏方法只保留最重要的1万块，其他99万块直接扔掉——反正它们的影响微乎其微。
