# PM2.5 CMAQ融合方法十折交叉验证架构文档

> **重要**：本文档定义了融合方法十折交叉验证的核心架构，所有方法验证必须遵循此架构，以确保创新性量化的正确性。
>
> **最后更新**：2026-04-15（新增三种训练坐标方式对比及推荐）

---

## 一、验证目的

十折交叉验证的目的是**评估方法对未知监测站点的预测能力**，而非生成完整的网格产品。

```
验证目的：筛选出最好的融合方法
最终应用：生成全区域高分辨率PM2.5网格产品
```

---

## 二、核心架构原则

### 2.1 标准验证流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        十折交叉验证流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【训练阶段】                                                    │
│  ┌──────────────┐                                               │
│  │ 9折监测站数据 │ → fit(训练集站点, 训练集CMAQ网格对应值)       │
│  └──────────────┘                                              │
│          ↓                                                     │
│  【预测阶段】                                                    │
│  ┌──────────────┐                                              │
│  │ 1折站点所在网格坐标 │ → predict(网格坐标) → 网格预测值        │
│  └──────────────┘                                              │
│          ↓                                                     │
│  【评估阶段】                                                    │
│  ┌──────────────┐                                              │
│  │ 网格预测值 vs 1折站点真实值 │ → R², MAE, RMSE, MB          │
│  └──────────────┘                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**注意**：预测目标是验证站点**所在的CMAQ网格坐标**，而非站点本身。

### 2.2 关键原则

| 原则 | 说明 |
|------|------|
| **直接预测** | 训练完成后，直接对验证站点所在的CMAQ网格坐标进行预测 |
| **信息对等** | 训练时用什么信息，验证时也用对应信息 |
| **无需绕路** | 不需要先预测全网格再提取验证站点 |

### 2.3 错误做法（禁忌）

```
❌ 错误做法1：绕路到网格
   fit(训练集) → 全网格预测 → 提取验证站点 → 对比
   原因：标准方法不需要，引入不必要的复杂度

❌ 错误做法2：信息不对等
   训练用全网格CMAQ数据，验证却只检查站点
   原因：训练和预测的信息不一致，评估不公正

❌ 错误做法3：用网格评估替代站点验证
   原因：网格评估的是产品质量，站点验证的是方法能力
```

## 二（续）、训练坐标的三种方式

针对融合方法（如RK-Poly）的GPR残差克里金训练，坐标可以选择三种方式：

### 方式1：监测站坐标

```
训练坐标：监测站 (Lon, Lat) 坐标
预测坐标：1折站点所在的CMAQ网格坐标

特点：
- 训练时使用原始监测站坐标，样本量完整（~1300个训练点）
- 预测时使用CMAQ网格坐标
- 同一CMAQ网格可能对应多个监测站（坐标不同）
```

### 方式2a：CMAQ网格坐标（聚合）

```
训练坐标：监测站所在的CMAQ网格坐标（聚合）
预测坐标：1折站点所在的CMAQ网格坐标

特点：
- 训练时将同一CMAQ网格内的多个监测站取均值
- 训练坐标唯一，无重复
- 样本量减少约一半（~525个训练点）
- 聚合损失了网格内站点间的真实残差差异
```

### 方式2b：CMAQ网格坐标（不聚合）【推荐】

```
训练坐标：监测站所在的CMAQ网格坐标（不聚合）
预测坐标：1折站点所在的CMAQ网格坐标

特点：
- 训练时使用CMAQ网格坐标，但保留所有记录（坐标可能重复）
- 同一CMAQ网格可有多条记录，对应不同残差值
- 样本量完整（~1300个训练点）
- 实践验证效果最优（4天测试：R²=0.8762, RMSE=15.36, |MB|=0.03）
```

### 对比示意

```
监测站 → get_cmaq_grid_coord()
    │
    ├─ 方式1: X_train = [(Lon₁,Lat₁), (Lon₂,Lat₂), ...]      # ~1429个坐标
    ├─ 方式2a: X_train = [(CMAQ_lon₁,CMAQ_lat₁), ...]        # 聚合后~525个坐标
    └─ 方式2b: X_train = [(CMAQ_lon₁,CMAQ_lat₁), (CMAQ_lon₁,CMAQ_lat₁), ...]  # ~1429条记录（重复坐标）
```

### 四天对比测试结果（2020-01-01 ~ 01-04）

| 方式 | 平均R² | 平均RMSE | 平均|MB| |
|------|--------|---------|--------|
| 方式1 | 0.8701 | 15.72 | 0.21 |
| 方式2a | 0.8386 | 17.69 | 0.41 |
| **方式2b** | **0.8762** | **15.36** | **0.03** |

### 推荐原则

| 方法类型 | 推荐训练坐标 | 说明 |
|----------|-------------|------|
| **所有GPR克里金类方法** | **方式2b： CMAQ网格坐标（不聚合）** | 实践验证效果最优，R²最高，|MB|最小 |
| 备选 | 方式1：监测站坐标 | 效果接近方式2b，可作为备选 |

**统一使用方式2b（不聚合）作为GPR克里金类方法的训练坐标**

## 三、两种验证模式

### 3.1 标准模式（适用于绝大多数方法）

```
训练输入：9折监测站数据 + 9折站点对应的CMAQ网格值
预测目标：1折监测站所在的CMAQ网格坐标

示例方法：
- RK-Poly（Residual Kriging with Polynomial）
- SPIN-Kr（图核时空克里金）
- BMSF-Geostat（贝叶斯多源融合）
- CorrDiff-3km（残差修正扩散降尺度）
- VNA / eVNA / aVNA
```

**验证逻辑**：
```python
# 十折循环
for fold_id in range(1, 11):
    train_df = 9折数据
    test_df = 1折数据

    # 训练 - 仅用训练集
    model.fit(train_df[['Lon', 'Lat']].values, train_df['Conc'].values,
              train_df['CMAQ'].values if needed)

    # 预测 - 对验证站点所在的CMAQ网格坐标预测
    pred = model.predict(test_df[['CMAQ_grid_Lon', 'CMAQ_grid_Lat']].values)

    # 评估
    metrics = compute_metrics(test_df['Conc'].values, pred)
```

### 3.2 特例模式（必须预测全网格）

```
适用条件：该方法在训练时依赖全网格范围，输出结果受网格边界影响

特征：
- 训练过程中使用全模型域数据
- 局部区域预测结果与全区域预测结果不同
- 存在边界效应

示例：
- Downscaler（MCMC降尺度方法）
```

**判断依据：**

| 问题 | 标准模式 | 特例模式 |
|------|----------|----------|
| 训练是否依赖全网格数据？ | 否，仅用站点数据 | 是，需要全网格 |
| 预测是否有边界效应？ | 否，局部=全局 | 是，边界影响结果 |
| 是否可以直接对站点网格坐标预测？ | 是 | 否，必须预测全网格 |

**选择流程：**
```
方法训练是否依赖全网格数据？
├── 否 → 标准模式
└── 是 → 特例模式（Downscaler等）
```

**验证逻辑**：
```python
for fold_id in range(1, 11):
    train_df = 9折数据
    test_df = 1折数据

    # 训练 - 用全网格信息
    model.fit(train_df, full_grid_cmaq_data)

    # 预测 - 必须预测全网格
    full_grid_pred = model.predict(full_grid_coords)

    # 提取验证站点位置的值
    pred = extract_at_stations(full_grid_pred, test_df[['Lon', 'Lat']])

    # 评估
    metrics = compute_metrics(test_df['Conc'].values, pred)
```

### 3.3 两种模式对比

| 模式 | 训练输入 | 预测目标 | 代表方法 |
|------|----------|----------|----------|
| **标准模式** | 9折站点 | 1折站点所在的CMAQ网格坐标 | RK-Poly, SPIN-Kr, BMSF-Geostat |
| **特例模式** | 9折站点 + 全网格 | 全网格（必须） | Downscaler |

## 四、VNA / eVNA / aVNA 验证要点

### 4.1 方法回顾

| 方法 | 公式 | 物理含义 |
|------|------|----------|
| **VNA** | VNA = Σ(w_i × obs_i) | 监测值空间插值（Voronoi邻居IDW） |
| **aVNA** | aVNA = M + bias, bias = obs - M | 加法修正 |
| **eVNA** | eVNA = M × r_n, r_n = obs / M | 乘法修正 |

### 4.2 正确验证流程

VNA/eVNA/aVNA应该采用**标准模式**，对验证站点所在的CMAQ网格坐标预测：

```python
# 当前正确实现（标准模式）
for fold_id in range(1, 11):
    train_df = 9折数据
    test_df = 1折数据

    # NNA拟合
    nn = NNA(method='voronoi', k=30, power=-2)
    nn.fit(train_df[['Lon', 'Lat']],
           train_df[['Conc', 'mod', 'bias', 'rn']])

    # 对验证站点所在的CMAQ网格坐标预测
    # 注意：验证站点位于某个CMAQ网格上，预测的是该网格位置的值
    zdf = nn.predict(test_df[['CMAQ_grid_Lon', 'CMAQ_grid_Lat']].values)
    vna_pred = zdf[:, 0]
    vna_bias_pred = zdf[:, 2]
    vna_rn_pred = zdf[:, 3]

    # 计算融合结果
    aVNA_pred = test_df['CMAQ'].values + vna_bias_pred
    eVNA_pred = test_df['CMAQ'].values * vna_rn_pred

    # 评估
    ...
```

## 五、Downscaler 验证要点

### 5.1 Downscaler 特殊性

Downscaler 属于**特例模式**，原因：
- 训练时使用全 CMAQ 网格数据
- 输出结果依赖网格范围（边界效应）
- 无法仅用站点数据进行公正评估

### 5.2 验证流程

```python
for fold_id in range(1, 11):
    train_df = 9折数据
    test_df = 1折数据

    # 获取全网格CMAQ数据
    full_grid_cmaq = load_full_cmaq_grid()

    # 训练
    downscaler.fit(
        train_df[['Lon', 'Lat']].values,
        train_df['Conc'].values,
        full_grid_cmaq
    )

    # 必须预测全网格
    full_pred = downscaler.predict(full_grid_coords)

    # 提取验证站点位置
    pred = extract_at_stations(full_pred, test_df[['Lon', 'Lat']].values)

    # 评估
    metrics = compute_metrics(test_df['Conc'].values, pred)
```

### 5.3 与标准模式的区别

| 步骤 | 标准模式 | Downscaler（特例） |
|------|----------|-------------------|
| 训练数据 | 仅9折站点 | 9折站点 + 全网格 |
| 预测目标 | 1折站点坐标 | 全网格（必须） |
| 提取步骤 | 无需提取 | 必须从网格提取 |

## 六、验证指标

### 6.1 核心指标

| 指标 | 公式 | 含义 |
|------|------|------|
| **R²** | 1 - Σ(y-ŷ)² / Σ(y-ȳ)² | 决定系数，越接近1越好 |
| **MAE** | Σ|y-ŷ| / n | 平均绝对误差，越小越好 |
| **RMSE** | √(Σ(y-ŷ)² / n) | 均方根误差，越小越好 |
| **MB** | Σ(ŷ-y) / n | 平均偏差，正值表示高估 |

### 6.2 指标计算代码

```python
def compute_metrics(y_true, y_pred):
    """计算评估指标"""
    mask = ~(np.isnan(y_true) | np.isnan(y_pred))
    y_true = y_true[mask]
    y_pred = y_pred[mask]

    if len(y_true) == 0:
        return {'R2': np.nan, 'MAE': np.nan, 'RMSE': np.nan, 'MB': np.nan}

    return {
        'R2': r2_score(y_true, y_pred),
        'MAE': mean_absolute_error(y_true, y_pred),
        'RMSE': np.sqrt(mean_squared_error(y_true, y_pred)),
        'MB': np.mean(y_pred - y_true)
    }
```

## 八、常见问题

### Q1: 为什么十折验证用站点而不是网格？

**A**：十折验证的目的是评估方法对**未知数据的预测能力**。站点数据有真实值可以对比，网格数据没有。网格评估是最终产品的质量评估，与方法验证是不同层面的问题。

### Q2: VNA/eVNA/aVNA 为什么不需要预测全网格？

**A**：因为 VNA/eVNA/aVNA 是基于站点插值的方法，fit 和 predict 都只需要站点坐标信息（具体是站点所在的CMAQ网格坐标），不依赖全网格数据。所以直接对验证站点所在的CMAQ网格坐标预测即可。

### Q3: 什么情况下必须预测全网格？

**A**：只有当方法在训练或预测时**依赖全网格范围**，输出结果**受网格边界影响**时，才需要预测全网格。典型例子是 Downscaler（有 MCMC 采样边界效应）。

### Q4: 如果方法既可以用标准模式又可以用特例模式怎么办？

**A**：以**方法原始设计**为准。如果方法论文中描述的是全网格产品，则属于特例；如果描述的是站点校正，则用标准模式。

## 九、代码模板

### 9.1 标准模式模板

```python
def ten_fold_standard(method_class, method_name, selected_day):
    """
    标准模式十折交叉验证

    适用于：RK-Poly, SPIN-Kr, BMSF-Geostat, CorrDiff-3km, VNA/eVNA/aVNA
    """
    # 加载数据
    monitor_df = pd.read_csv(monitor_file)
    fold_df = pd.read_csv(fold_file)

    day_df = monitor_df[monitor_df['Date'] == selected_day].copy()
    day_df = day_df.merge(fold_df, on='Site', how='left')
    day_df = day_df.dropna(subset=['Lat', 'Lon', 'Conc'])

    all_y_true = []
    all_y_pred = []

    for fold_id in range(1, 11):
        train_df = day_df[day_df['fold'] != fold_id]
        test_df = day_df[day_df['fold'] == fold_id]

        # 训练
        model = method_class(**params)
        model.fit(train_df[['Lon', 'Lat']].values,
                  train_df['Conc'].values,
                  train_df['CMAQ'].values if needed)

        # 对验证站点所在的CMAQ网格坐标预测
        pred = model.predict(test_df[['CMAQ_grid_Lon', 'CMAQ_grid_Lat']].values)

        all_y_true.extend(test_df['Conc'].values)
        all_y_pred.extend(pred)

    return compute_metrics(np.array(all_y_true), np.array(all_y_pred))
```

### 9.2 特例模式模板（Downscaler）

```python
def ten_fold_downscaler(method_class, method_name, selected_day):
    """
    特例模式十折交叉验证

    适用于：Downscaler（必须预测全网格）
    """
    # 加载全网格CMAQ数据
    cmaq_grid = load_full_cmaq_grid()
    grid_coords = get_grid_coords()

    all_y_true = []
    all_y_pred = []

    for fold_id in range(1, 11):
        train_df = day_df[day_df['fold'] != fold_id]
        test_df = day_df[day_df['fold'] == fold_id]

        # 训练（全网格）
        model = method_class(**params)
        model.fit(train_df, cmaq_grid)

        # 必须预测全网格
        full_pred = model.predict(grid_coords)

        # 提取验证站点位置
        pred = extract_at_stations(full_pred, test_df[['Lon', 'Lat']].values)

        all_y_true.extend(test_df['Conc'].values)
        all_y_pred.extend(pred)

    return compute_metrics(np.array(all_y_true), np.array(all_y_pred))
```

## 九（续）、分阶段验证流程

### 9.1 四阶段验证体系

创新方法验证采用**分阶段递进**策略：

| 阶段 | 日期范围 | 天数 | 用途 |
|------|----------|------|------|
| **pre_exp** | 2020-01-01 ~ 2020-01-05 | 5天 | 快速预验证 |
| **stage1** | 2020-01-01 ~ 2020-01-31 | 31天 | 冬季全面验证 |
| **stage2** | 2020-07-01 ~ 2020-07-31 | 31天 | 夏季梅雨验证 |
| **stage3** | 2020-12-01 ~ 2020-12-31 | 31天 | 冬季雾霾验证 |

### 9.2 VNA 基线阈值

各阶段的基准方法VNA必须达到的最低指标：

| 阶段 | R² | RMSE | MB |
|------|-----|------|-----|
| **pre_exp** | ≥ 0.8941 | ≤ 16.42 | ≤ 0.76 |
| **stage1** | ≥ 0.9057 | ≤ 16.28 | ≤ 0.50 |
| **stage2** | ≥ 0.8458 | ≤ 4.97 | ≤ 0.04 |
| **stage3** | ≥ 0.9078 | ≤ 11.90 | ≤ 0.36 |

### 9.3 创新判定标准

创新方法必须**同时满足**三个条件：

| 指标 | 要求 | 说明 |
|------|------|------|
| **R²** | > 基线 + 0.01 | 提升必须显著 |
| **RMSE** | ≤ 基线 | 不能变差 |
| **\|MB\|** | ≤ \|基线\| | 偏差不能超过基线 |

### 9.4 分阶段执行流程
