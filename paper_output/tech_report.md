## 二、方法通俗解读

### 2.1 为什么要融合CMAQ和监测数据？

**CMAQ模型**：像一张高分辨率地图，能告诉你北京每个角落的PM2.5浓度，但这条"地图"可能有系统性偏差（比如总是低估20%）。

**监测站数据**：准确但稀疏，只有1400多个点，无法覆盖全区域。

**融合的目标**：让CMAQ的高分辨率 + 监测站的准确性 结合，产生一张既准确又覆盖全区域的数据。

### 2.2 RK-Poly的"分解-组合"策略

**类比**：预测明天考试分数
1. 你有一个"历年规律"（多项式校正）：告诉你在80分水平你应该得多少
2. 你参考周围同学估分（空间插值）：他们各自估了多少，综合起来修正你的分数
3. 最终预测 = 历年规律 + 同学修正

**三步走**：
```
Step 1: 多项式校正
  CMAQ说某地80 → 用历年规律校正为96 (80×1.2)

Step 2: 空间残差插值
  周围站点实际比CMAQ高15，但这个站点可能只高10
  
Step 3: 融合
  最终 = 96 + 10 = 106
```

### 2.3 消融实验解读

| 方法 | R² | 通俗解释 |
|------|-----|---------|
| CMAQ原始 | -0.04 | CMAQ直接用，效果很差 |
| BC_Mean | 0.75 | 假设偏差是常数，一律加15 |
| BC_Scale | 0.79 | 偏差与浓度成正比，高时多加点 |
| OLS | 0.80 | 偏差是线性关系 |
| RK-OLS | 0.83 | OLS后加上空间插值修正 |
| **RK-Poly** | **0.85** | 用曲线（非线性）校正 + 空间插值 |

**结论**：高污染时CMAQ低估更严重（二次项），空间插值能捕捉局部偏差

### 2.4 为什么有些论文方法效果差？

| 方法 | R² | 原因分析 |
|------|-----|---------|
| FC1/FC2/FCopt | 0.02~0.36 | 设计针对北欧平坦地形，北京地形复杂不适用 |
| Universal Kriging | 0.58 | 假设偏差与土地利用线性关系，实际更复杂 |
| Bayesian DA | 0.42 | 先验分布假设不适合北京数据 |
| HDGC | 0.49 | 方法假设与实际数据特征不匹配 |

**教训**：没有万能方法，每个方法的假设必须与实际数据特征匹配。

## 五、RK-Poly方法核心公式（真正有效的创新方法）

$$P_{RK-Poly} = \underbrace{a + bM + cM^2}_{\text{多项式校正}} + \underbrace{R^*_{GPR}}_{\text{空间插值残差}}$$

### 步骤

1. **多项式OLS校正**: $O = a + bM + cM^2 + e$ (二次多项式)
2. **GPR残差建模**: 对残差$e$使用RBF核高斯过程回归进行空间插值
3. **融合预测**: $P = a + bM + cM^2 + GPR(e)$

**为什么RK-Poly是有效创新**：
- 二次项捕捉 CMAQ 在高污染时的非线性偏差
- GPR 空间插值修正局地偏差
- 参数相对稳定（有物理机制）

## 九-B、参数迁移性：方法论设计的核心矛盾

### 核心问题

| 方案 | 问题 |
|------|------|
| 只跑一天预实验调参 | 参数只对那一天有效，其他天用同样参数结果变差 |
| 从第一天就跑全部天数 | 太贵，失去了预实验的意义 |
| 物理意义赋值 | 很多参数（尤其是偏差校正系数）根本没有物理意义可以查表 |

### 两阶段参数策略

```
第一阶段：物理初始化
↓
用物理意义给参数赋"合理初值"
（比如 Kriging 的 range 初值，可以用 PM2.5 典型空间相关距离）
↓
第二阶段：有限数据验证
↓
在 1-2 天上做参数微调
→ 验证"这套初值 + 少量数据 → 能泛化到其他天"
↓
跨天测试（看参数是否迁移）
```

### 四元数组的隐性判断维度

| 维度 | 评判标准 |
|------|---------|
| 1.定量提升 | R² ≥ 历史最佳 + 0.01 |
| 2.物理可解释性 | 参数能用物理意义初始化，或在有限数据下稳定收敛 |
| 3.跨日期一致性 | 在≥3个不同天测试，参数不剧烈震荡 |

**第2条的重点**：不是"参数完全来自物理"，而是"**参数有物理锚点，不怕小数据扰动**"

### 关键洞见：参数迁移性的本质

> **参数对某一天的数据特别敏感**（小扰动导致大变化）→ 这个方法本质上是在"拟合那一天"，不是"建模"
> **参数对不同天都稳定** → 这个方法是真的有物理基础

### 这条规则能排除的方法

| 类型 | 特征 | 为什么会被排除 |
|------|------|---------------|
| 数据饥渴型 | 依赖大量数据才能拟合，数据少了就崩 | 论文会被审稿人挑战 |
| 日期波动型 | 参数在好日期和坏日期之间剧烈波动 | 说明本质上是在拟合日期特征 |

### CSP-RK 的失败本质

CSP-RK 通过浓度分层（T1=35, T2=75 μg/m³）为**每天独立拟合** OLS 参数：

- 单日预实验：R²=0.8535 vs RK-Poly=0.8519，Δ=+0.0016 ✅
- 1月整月验证：平均 Δ≈-0.00003 ❌

**结论**：分层校正参数对日期敏感，本质上是"拟合日期特征"而非"建模"，在跨日期验证中无法保持一致性。
